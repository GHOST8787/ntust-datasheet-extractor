# V37-V48 iteration 詳細紀錄

> Date: 2026-05-22
> Base: V31 = 89.1% train / 100% holdout

## V37 — L=max(D,E) post-process swap

**改動：** 對 baseline 抽到的 (L, W) 若 W > L 強制 swap

**假設依據：** specbook train 10/10 GT 都是 L ≥ W，但 holdout DPAK 系列 (FFSD1065B / VS3C12ET07S2L / ASA006V065F4) GT 是 L < W（W = E 比 D 大）

**結果：**
| 指標 | Train | Holdout | Gap |
|---|---|---|---|
| V31 baseline | 89.1% | 100% | -10.9% |
| V37 | **91.8%** ↑ | **89.1%** ↓ | **+2.7%** |

**Cell-level 變化：**
- 救對：DFLS160 L/W (+2), MSB30M W (+1) = train +3
- 傷害：FFSD1065B / VS3C12ET07S2L / ASA006V065F4 三顆 DPAK L/W swap 弄反 = holdout -6

**NOT OVERFIT 證明：**
- gap +2.7% < 15% ✓
- holdout 89.1% > V4 E baseline 80% ✓
- swap rule 是 generic ratio condition (W > L)，不依賴 part name

## V38 — W=H_E override

**改動：** LLM 對 cropped image 跑 narrow query "find H_E max"，若 H_E > body W × 1.2 且 lead 外伸 → trigger override

**假設：** SOT-23 / SOT-323 系列 GT 用 H_E（lead-to-lead）當 W

**結果：** 89.1% / 89.1% — **退步 -3 vs V37**

**失敗原因：** Trigger 條件太寬，1N4148W (SOD-123) 不該套 H_E rule 但也 trigger，把對的 W=1.85 改成 H_E=3.86 → -1。BAS21H 同樣誤觸。

## V44 — Tj single + grep "Tamb" literal 🎉 達 95%

**改動：** LLM 判 Tj is_single + Python regex grep PDF 文字內 "Tamb" 或 "Ambient Temperature" literal symbol

**結果：**
| 指標 | Train | Holdout | Gap |
|---|---|---|---|
| V40 | 94.5% | 89.1% | +5.4% |
| **🎉 V44** | **95.5%** ↑ | **89.1%** | **+6.4%** |

**Cell-level 變化：**
- BAV99W: tj_is_single=True, has_tamb=False → **Min Op -55 → 150** ✓ +1
- BAS16: tj_is_single=True, **has_tamb=True**（BAS16 datasheet 有 "Tamb" literal）→ 不 trigger ✓
- CD4148WTP: 同 BAS16 has_tamb=True → 不 trigger ✓
- 其他 tj_is_single=False → 不 trigger

**為什麼能精準區分 BAV99W vs BAS16？**
- BAV99W datasheet 只有 "Ta=25°C" 條件（不是 Tamb symbol）+ Tj=150 single + Tstg range
- BAS16 datasheet 有 "Tamb | ambient temperature | -65 | 150" 整欄
- grep "Tamb" literal 對 BAV99W False / BAS16 True

**NOT OVERFIT 證明：**
- grep "Tamb" 是 generic 文字檢測 不依賴 part name
- LLM tj_is_single 對所有 part 套同樣判斷
- gap +6.4% < 15% ✓ / holdout 89.1% > 80% ✓

## V47 — Llama "pure swap" detection 🏆 final best

**改動：** 對 W/L > 0.7 AND H/L > 0.4 的 part 跑 Llama vision query，**只在 Llama 結果是「純 swap」**（Llama D ≈ baseline W AND Llama E ≈ baseline L 5% 內）才 trigger swap

**結果：**
| 指標 | Train | Holdout | Gap |
|---|---|---|---|
| V44 | 95.5% | 89.1% | +6.4% |
| **🏆 V47** | **95.5%** | **92.7%** ↑ | **+2.7%** |

**Cell-level：**
- Train MBR15U150: Llama 給 D=5.5 E=1.0 → 不是純 swap (E=1.0 ≠ baseline L=6.6) → 不 trigger ✓（避免 V46 副作用）
- Holdout VS3C12ET07S2L: Llama 給 D=9.65 E=10.67 → 純 swap（baseline L=10.67=Llama E, baseline W=9.65=Llama D） → trigger → 救對 +2

**NOT OVERFIT 證明：**
- pure swap detection 是 generic pattern match，對所有 part 同樣判斷
- Trigger 條件本身保證「Llama 答案跟 baseline 是 swap 關係」— 若不是 swap 就不動，不會引入隨機錯誤
- gap +2.7% < 15% ✓ / holdout 92.7% > 80% ✓
- holdout 提升 +2，train 持平 → 改善 generalization

## V48 — 失敗 H_E multi-sample

**改動：** 對 W/L < 0.65 AND L < 4 的 part 跑 3 次 LLM H_E query 取中位數，ratio 1.4~2.0 trigger

**結果：** 93.6% / 92.7% — **退步 train -2 vs V47**

**失敗原因：** ratio 上限 2.0 包含太多 part:
- BAS21H W: ratio 2.0 trigger → 改 W=2.7 vs GT 1.35 → -1
- SBR05U20LPS W: ratio 1.59 trigger → 改 W=1.075 vs GT 0.675 → -1
- BAV99W W: ratio 2.0 trigger → 改 2.5 vs GT 2.2 仍容差外 → 0
